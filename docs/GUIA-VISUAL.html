0 Regra de ouro: subscription, não API

  • Claude Code logado na conta Anthropic (Pro/Max/Team)

  • Grok app / Grok Build / xAI sub
  • Gemini app Google / Gemini CLI / AI Pro — ver GEMINI.md

NÃO USE (API paga de AGENT)

  • ANTHROPIC_API_KEY pay-as-you-go para “ser o Claude”
  • OPENAI_API_KEY avulsa para coding agent
  • GEMINI_API_KEY / AI Studio pay-as-you-go como motor do agent
  • OpenRouter como motor do agente (≠ modelo Gemini no OpenRouter do app)
  • Scripts que disparam LLM pago em background pro agent

Exceção: keys OpenRouter do produto (tabela abaixo).

Duas carteiras — agent na sub; OpenRouter só no produto. Agent e app leem OPEN_ROUTER_API_KEY da env do sistema — não pedem a key no chat.

Tabela canônica — keys

Key Agent faz o quê? Para quê

7 Produto WaifuChat ≠ agente

OpenRouter no app = o agent pode pedir essa key ao configurar o produto.
Fluxo típico do agent:
  • Agent não pede a key — lê OPEN_ROUTER_API_KEY do sistema
  • Smoke: GET /api/health
  • Se faltar: HANDOFF “setar no Windows” (sem colar secret no chat)
  • Nunca commitar; nunca usar como motor do coding agent
UI/personas/paywall mock podem ir sem key; smoke de stream = precisa OpenRouter do app.
